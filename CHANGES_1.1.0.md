# NO MORE BETS 1.1.0 修改清单

本次版本目标：一次性补齐正式上架前需要的登录入口、管理员模式、订阅绑定、安全、数据统计、日期、账号删除与审核素材，不破坏原有可用功能。

## 已修改

### 账号与登录
- 新增登录入口：未登录时先进入登录页，已登录后进入主 App。
- 支持邮箱/昵称创建本地账号。
- 支持访客模式，保留低门槛使用体验。
- 新增统一 AuthProvider，管理登录状态、当前用户、管理员角色、退出登录、删除账号。
- 登录后绑定 RevenueCat App User ID，避免订阅状态和匿名用户混乱。

### 管理员模式
- 新增管理员中心页面 `src/app/admin.tsx`。
- 仅管理员账号显示管理员入口。
- 管理员中心默认只展示本机聚合数据，不直接展示用户私人日记、联系人电话、未来信等敏感内容。
- 预留后端角色控制说明：正式云端版本应由服务器/数据库判断管理员角色。

### 订阅与付费
- 统一 RevenueCat entitlement 配置到 `src/config.ts`。
- Paywall 文案更新为“3天免费试用，之后通过 App Store 自动续订”。
- 保留恢复购买。
- AI 倾诉入口增加 Pro 权限检查，未订阅时弹出付费墙。

### 安全与 AI
- 删除前端硬编码 Groq API Key。
- 删除前端直接请求 Groq API 的代码。
- 新增 `AI_PROXY_URL` 配置位：正式 AI 服务必须通过自己的后端代理访问，API Key 不得放入 App 前端。
- AI 代理未配置时，保留本地冲动急救回复，不影响用户在危险时刻使用基础支持。
- 新增危机关键词保护：出现自伤、伤人、严重危机时，引导用户拨打 988、911 或联系真人。

### 数据与日期
- 统一日期为本地日期，避免 UTC 导致美国东部晚上记录跑到第二天。
- 将每日记录作为统计唯一数据源。
- 修复 streak、longestStreak、monthlyDays、monthlyLoss、todayChecked、todayGambled 的重复计算/错算风险。
- 修复日历点击旧日期后不加载该日期旧记录的问题。
- 修复编辑旧记录可能影响当前月份统计的问题。

### 个人页面
- 增加账号卡片。
- 增加退出登录。
- 增加删除账号与本机数据。
- 增加隐私政策、服务条款、联系支持入口。
- 热线电话改为可点击拨号。

### App 配置与清理
- App 版本升级到 1.1.0。
- iOS buildNumber 升级到 3。
- Android 移除未使用的 RECORD_AUDIO 权限。
- 删除未使用的 `@google/generative-ai` 依赖。
- 关闭 React Compiler 实验项，避免非必要严格规则影响当前 Expo 项目。

## 重要限制

当前项目没有接入后端数据库。本次为了不破坏原有可用功能，登录与管理员模式采用“本地账号 + RevenueCat 绑定 + 管理员入口 + 后端预留接口”的稳定方案。以后如果需要换手机同步、云端管理员后台、客服后台，应接 Supabase/Firebase/自有后端，再替换 `src/auth.tsx` 与数据层。

AI 现在不会泄露 API Key。若要恢复真实 AI 聊天，请先部署后端代理，并在 `src/config.ts` 填入 `AI_PROXY_URL`。

## 订阅完善追加修改（Build 3）

### RevenueCat 与订阅状态
- 新增 `configureRevenueCat()`，避免多处重复初始化 RevenueCat。
- 新增订阅快照 `SubscriptionSnapshot`，统一读取 Pro 状态、entitlement、到期/续订日期、管理订阅链接和错误提示。
- `hasActivePro()` 支持配置的 entitlement，同时兼容常见 `pro/premium` entitlement 名称，降低后台命名不一致导致无法解锁的风险。
- 登录、退出、恢复购买前都会先确保 RevenueCat 已初始化。
- 个人页新增订阅状态卡：显示高级会员是否激活、到期/续订日期、恢复购买、管理订阅。

### Paywall 付费页
- 重做 Paywall 布局：顶部关闭、功能说明、年度/月度方案、主按钮、恢复购买、管理订阅、隐私/条款链接。
- 文案明确说明：3天免费试用、通过 Apple ID 付款、试用结束后自动续订、可在 Apple ID 订阅管理取消、删除 App/账号不会自动取消订阅。
- Paywall 支持传入 `featureName`，用户点击具体 Pro 功能时会说明该功能属于高级功能。
- 增加 iOS 环境判断：非 iOS 真机/TestFlight/App Store 环境显示订阅不可用提示，避免开发环境误判。
- 增强购买失败、恢复失败、订阅配置异常、网络异常的用户提示。

### Pro 功能软限制
- AI 倾诉继续作为高级功能保留。
- 免费用户可保留基础联系人和基础目标；添加更多联系人、联系人照片、更多目标时弹出 Paywall。
- 不删除用户已经创建的数据，避免升级后破坏原本可用功能。

### AI 聊天保留
- 保留 AI 聊天入口和 Pro 权限控制。
- 后端未配置时自动使用本机急救回复，不影响紧急场景。
- AI 后端异常时显示提示，并自动降级到本机急救回复。
