# Google Play 数据安全 (Data Safety) 填写映射 — NO MORE BETS

> Play Console → 政策 → App 内容 → 数据安全。下面是逐项建议答案，依据代码实际采集情况（对齐 Apple 隐私问卷）。
> 原则：**如实、从严**。App 采集的戒赌记录属于敏感个人信息，宁可多声明也不要漏报。

## 全局问题
- **你的 App 是否收集或共享用户数据？** → **是**
- **传输中是否加密？** → **是**（全部走 HTTPS：Supabase、AI 代理、RevenueCat）
- **用户能否请求删除数据？** → **是**。App 内路径：个人中心 → 隐私与安全 → 删除账号与本机数据（调用服务端 delete-account，彻底删除认证用户与云端行）。
- **数据是否遵循 Play 家庭政策 / 面向儿童？** → **否**，目标受众为成年人（见目标受众问卷，设 18+）。

## "收集 (Collected)" vs "共享 (Shared)"
- 全部第三方均为**代我处理的服务提供商**（Supabase = 自有后端；RevenueCat = 订阅状态；Sentry = 崩溃诊断）。按 Google 定义，服务提供商代处理**不算"共享"** → **共享一律选"否"**，但**收集要如实声明**。
- **不用于广告或第三方营销**；**不做用户画像广告**。

## 逐类型声明

| 数据类型 (Google 分类) | 是否收集 | 用途 (Purpose) | 是否必需 | 说明 |
|---|---|---|---|---|
| **个人信息 → 邮箱地址** | 是 | 账号管理、App 功能 | 必需 | 邮箱/Apple/Google 登录 |
| **个人信息 → 姓名** | 是 | App 功能 | 可选 | 用户填的昵称（可选） |
| **个人信息 → 其他信息** | 是 | App 功能 | 可选 | 戒赌相关：戒赌起始日、每日打卡、触发点、目标、重要联系人、给未来的信、**自报赌博损失金额** |
| **照片和视频 → 照片** | 是 | App 功能 | 可选 | 头像（相机/相册二选一，可不设） |
| **应用活动 → 其他用户生成内容** | 是 | App 功能 | 可选 | 励志墙投稿（提交后服务端审核，默认私有、不自动公开） |
| **应用信息和性能 → 崩溃日志** | 是 | App 功能（诊断） | 可选 | Sentry 崩溃上报；无 SENTRY_AUTH_TOKEN 时仅上报崩溃、不传源码映射 |
| **应用信息和性能 → 诊断信息** | 是 | App 功能（诊断） | 可选 | Sentry 性能/诊断 |

### 敏感性提示（重要）
- 「自报赌博损失金额 + 戒赌恢复记录」具有健康/成瘾属性。Google 数据安全没有"赌博"专类，上面归入**个人信息→其他信息**已足够；若审核追问，可补充说明这是**成瘾恢复自助数据，仅用于 App 功能、不外传、不用于广告**。
- **不收集**：精确/大致位置、通讯录、短信/通话、日历、金融账户/支付卡号（订阅走 Google Play，App 不接触卡号）、麦克风录音（已在 app.json 用 blockedPermissions 屏蔽 RECORD_AUDIO）。
